<Title>Styling</Title>

`JBTextarea` is the multiline counterpart to `JBInput`. It follows the same token model for borders, color, typography, focus, disabled, and invalid states, then adds textarea-specific height and slot surfaces for multiline composition.

## Common Design Tokens

Use the core theme docs as the source of truth for shared design-system variables:

- [Theme Colors](../../jb-core/theme/stories/colors.mdx) explains the general color palette variables, default colors, customization approaches, and color generation helpers.
- [Theme Sizes](../../jb-core/theme/stories/sizes.mdx) explains shared size and radius variables.
- [Theme Custom Galleries](https://javadbat.github.io/design-system/?path=/docs/theme-custom-material--docs) gathers the shared custom style recipes across components.

Textarea recipes should use those common tokens where possible, then map them into `--jb-textarea-*` variables only where the textarea needs a component-specific value.

## Styling Model

Use three layers when styling textarea examples:

- Common design tokens are documented in [Theme Colors](../../jb-core/theme/stories/colors.mdx) and [Theme Sizes](../../jb-core/theme/stories/sizes.mdx).
- Reusable example theme packs live in [`docs/styles`](https://github.com/javadbat/design-system/tree/main/docs/styles) and can be shared across components.
- Component recipe files live in [`modules/jb-textarea/stories/styles`](https://github.com/javadbat/design-system/tree/main/modules/jb-textarea/stories/styles).
- Public component variables use the `--jb-textarea-*` prefix and are the stable styling API.

Each example uses one shared example theme pack from [`docs/styles`](https://github.com/javadbat/design-system/tree/main/docs/styles) and one textarea-specific recipe from [`modules/jb-textarea/stories/styles`](https://github.com/javadbat/design-system/tree/main/modules/jb-textarea/stories/styles). The style story applies both classes directly to `JBTextarea`.

## Recommended Customization Path

1. Use `--jb-textarea-*` variables first for size, colors, typography, spacing, borders, shadows, and state values.
2. Match `jb-input` variables when the textarea should look like the multiline version of an input.
3. Use common theme variables from [Theme Colors](../../jb-core/theme/stories/colors.mdx) and [Theme Sizes](../../jb-core/theme/stories/sizes.mdx) when the style should align with the design system.
4. Put reusable theme-pack values in [`docs/styles`](https://github.com/javadbat/design-system/tree/main/docs/styles) only when they should be shared by multiple component demos.
5. Use `::part(...)` only for official exposed parts such as `::part(label)`, `::part(textarea-box)`, `::part(textarea)`, `::part(message)`, and slot wrappers.
6. Add a new component variable only when a needed internal surface cannot be customized with the current public variables or official parts.

## Theme File Pattern

Use [Theme Colors](../../jb-core/theme/stories/colors.mdx) and [Theme Sizes](../../jb-core/theme/stories/sizes.mdx) as the common design-token guides. Theme-pack files should keep reusable demo variables in [`docs/styles`](https://github.com/javadbat/design-system/tree/main/docs/styles), then each component should keep its own style CSS in its story [`styles`](https://github.com/javadbat/design-system/tree/main/modules/jb-textarea/stories/styles) folder.

```css
/* https://github.com/javadbat/design-system/blob/main/docs/styles/carbon.css */
.theme-name-style {
  /* shared design-system variables */
}

/* https://github.com/javadbat/design-system/blob/main/modules/jb-textarea/stories/styles/style-carbon.css */
.theme-name-textarea {
  /* textarea-specific variables and official parts */
}
```

This keeps common theme tokens reusable and lets each component page show only the CSS it needs.
